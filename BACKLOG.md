# Gridrunner Academy Backlog

Project-local backlog for Gridrunner Academy. Keep curriculum, lab, and kit
work here; only roll up cross-project dependencies to `workspace-tracker`.

## Current State

- Repository defines the academy concept and four initial project tracks:
  Signal Siege, Packet Quest, RF Ops, and Mesh Raiders.
- Licensing intent is documented: MIT for code/firmware and CC BY-NC-SA 4.0
  for educational content unless otherwise noted.

## High Priority

- Define the curriculum structure.
  - Create a standard lesson template with objective, safety boundary,
    hardware, setup, activity, reflection, and assessment sections.
  - Decide beginner/intermediate/advanced levels.
  - Add vocabulary and prerequisite sections for younger learners.

- Build the first complete lab.
  - Select one starter lab from Signal Siege or Packet Quest.
  - Keep the environment sandboxed and defensive.
  - Include setup, expected output, troubleshooting, and cleanup.

- Define hardware kit tiers.
  - Minimum no-solder kit.
  - ESP32/Raspberry Pi kit.
  - Optional RF exploration kit with clear legal/safety notes.
  - Include estimated cost and replacement parts.

- Create safety and ethics guidance.
  - Write clear boundaries for cybersecurity and RF activities.
  - Emphasize consent, local laws, and simulated environments.
  - Add instructor notes for supervising hands-on labs.

## Medium Priority

- Create project pages.
  - Signal Siege.
  - Packet Quest.
  - RF Ops.
  - Mesh Raiders.

- Add reusable assets.
  - Diagrams.
  - Printable checklists.
  - Lab worksheets.
  - Firmware or setup scripts where appropriate.

- Add a validation rubric.
  - Learner success criteria.
  - Instructor setup checklist.
  - Known failure modes and recovery steps.

## Validation

- Run the first lab from a fresh checkout.
- Confirm instructions work without hidden local assumptions.
- Review all cybersecurity and RF content for safe, authorized use.
- Confirm licenses are present before publishing reusable curriculum assets.
